# 💻 Programação Backend

## 🚀 Conceitos Fundamentais

* **Lógica de Programação e Algoritmo:** É a **base** para todas as linguagens de programação; estrutura essencial para a construção dos códigos.
* **Definição Backend:** Execução das funcionalidades existentes no **FRONT**, ou mais contextos para um **resultado factível**.
* **Tipos de Linguagens nos Dispositivos:** **Compilada** e **Interpretada**.
    * **Linguagem Compilada:** É convertida diretamente em um **código de máquina** que o processador pode executar.
    * **Linguagem Interpretada:** Executa o código diretamente como **streaming**, no momento da execução e é **traduzido**.

## 🗂️ Estruturas Essenciais

* **Variáveis:** Valor que pode ser **alterado** na execução do código (ex: BOOLEAN).
* **Variáveis Menos Utilizadas (exemplo):**
    * LONG INT
    * SHORT
    * UNION (C, C++)
    * ENUM: Valores numéricos inteiros
    * *1 BYTE = 8 BITS*
* **Operadores:** Lógicos, Relacionais e Aritméticos.
* **Estruturas Condicionais**
* **Laços de Repetição**
* **Orientação a Objetos**

## 🛠️ Funções e Bibliotecas

* **Funções:** Blocos de **códigos predefinidos** que podemos utilizar em nossos programas.
    * **Conceito em Python:** É uma parte ou bloco de código **pré-formatado** para executar uma determinada tarefa quando chamado no código principal.
    * **Premissa Essencial:** Não repetir nem reescrever código que já tinha sido feito, utilizando a função.
* **Bibliotecas (Libraries):** São um conjunto de **ferramentas** ou já **funções predefinidas** e amplamente **testadas** para utilizar nos programas extensivamente.
    * **Propósito:** Resolver um problema, **otimizar tempo**, utilizar códigos amplamente testados.
* **Framework:** Um conjunto de **instruções e bibliotecas** capazes de dar embasamento para vários códigos e **eliminando tempo de desenvolvimento** nos programas e projetos.
    * *Exemplo:* **.NET FRAMEWORK, MICROSOFT V 4.8.1**
* **Nomear Funções:** Nomear funções próprias com **exclusividade**, visando **não colidir** com as já existentes.
* **Funções Nativas do Python (70 Nativas):**
    1.  Manipulações de listas e sequências (remove, count, pop, insert)
    2.  Entrada e Saída (print e input)

## 🌐 Linguagens para Backend

* **Python (1986):**
    * Criador: **Guido van Rossum**
    * Crescimento das **Bibliotecas**
    * Código **aberto** com contribuições entre os desenvolvedores
    * Funções são um dos **pontos fortes** do Python, vista a vasta quantidade de comunidade de programadores que trabalham com essa linguagem de programação.
* **C#:** Linguagem de **Alto Nível** e **Orientada a Objetos**.
* **Java:** Compilado e utilizado majoritariamente pelos **bancos** e suas **aplicações**. Foco em **Segurança** e **Robustez**.
* **PHP - Preprocessor Hypertext:** **Opensource** com afinidade com **páginas WEB**. **Gratuito**.
* **JavaScript (1995):** Mais usada em **FRONTEND (WEB)**, mas também **BACKEND** com a mesma sintaxe.
* **Swift (2014):**
    * Apple
    * Mercado **nichado**
* **C++:** Linguagem padrão **TOP/DOWN**. Ótimo para o **desenvolvimento de jogos**.
* **Linguagens de Baixo Nível:**
    * **COBOL (1959) - COMON BUSINESS ORIENTED LANGUAGE**
    * **ASSEMBLY (1949)** - Escrito em **cartões perfurados**.
