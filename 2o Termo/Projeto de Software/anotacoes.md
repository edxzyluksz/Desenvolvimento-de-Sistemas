# Projeto de Software

Projeto - É algo que será desenvolvido para suprir uma demanda ou uma necessidade 

**Como começar um bom projeto**
- Definir claramente os stakeholders e suas expectativas, tal como verificar a viabilidade e ter a possibilidade de comunicação e importante documentar todas as etapas, planejamentos monetários e feedbacks 

**Ciclo de vida dos projetos** 
- Um projeto possui um ciclo de vida que se estende de seu início até seu encerramento. Um bom projeto pode aumentar sua vida útil dado ao bom planejamento inicial, execução, monitoramento e controle constantes em sua duração ao se tornar obsoleto = fim. 

**Tripla Restrição**
- A tripla restrição de um projeto se trata da relação: Escopo; Tempo; Custo, com questionamentos apropriados para a linha de reflexão:
  
  - Escopo: O objetivo do projeto é relevante para o mercado? Trará benefícios significativos em seu lançamento? 
  - Tempo: Dado às tendências atuais e saúde dos desenvolvedores, será possível terminar o projeto no prazo? 
  - Custo: Apesar de escopo bem definido e tempo viável, o quão caro será? Consumirá recursos proporcionais aos benefícios? 

As respostas deste triângulo devem ser positivas em todos para aumentar as chances do projeto não prejudicar a empresa ou deixar de ser descartado. De qualquer maneira, o ideal é equalizar todos os itens acima para compensar a escassez ou excesso de outro.

Por que ir para marte? 

- Pesquisa científica (aumento)

**O que abordar num projeto:** Tudo que é necessário para ser bem realizado. Deve existir verificação e amplo espectro para a realização. 

**Estudo da viabilidade:** Verificação se o projeto vai ser viável ou não. Se existe a possibilidade e capacidade de prosperar 

**Ciclo de vida:** Projeto é para funcionar durante o tempo determinado para ele. Evita falhas e travamentos.
