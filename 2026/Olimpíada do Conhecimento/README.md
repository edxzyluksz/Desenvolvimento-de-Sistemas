# Mobile - Processo Seletivo: O Desafio Prático

## DIOGO TAKAMORI BARBOSA


Nesta etapa, os alunos terão 3dias para entregar uma aplicação funcional. O foco não é um app gigante, mas um código limpo e bem estruturado.
Briefing do Desafio: "MyDailyTasks"

Objetivo: Desenvolver um aplicativo de gerenciamento de tarefas simples (To-Do List) com persistência de dados local.

Requisitos Funcionais (Obrigatórios):
- Tela de Listagem - Exibir todas as tarefas cadastradas.
- Tela de Cadastro - Permitir adicionar uma nova tarefa com: Título, Descrição e Nível de Prioridade (Baixa, Média, Alta).
- Funcionalidade de Conclusão - Permitir marcar a tarefa como "Feita" (Check).
- Exclusão - Permitir deletar uma tarefa (Swipe ou botão)

Requisitos Não-Funcionais (Diferenciais):
- Persistência: Os dados devem ser salvos localmente (Room, SQLite, SharedPreferences ou Hive, Json). O app não pode perder dados ao ser fechado.
- Stack:  a linguagem que o aluno dominar.
- UI/UX: Layout limpo, uso de ícones, cores consistentes (Material Design ou Cupertino).
- Arquitetura: Uso de MVC, MVP ou MVVM (demonstra organização).

Entregável:
- Link do Protótipo no Figma ou Canva
- Link do repositório no GitHub (O código deve ter commits que mostrem a evolução, não apenas um "upload final").
- Um vídeo curto (max 2 min) do aluno navegando no app e explicando uma parte do código.
